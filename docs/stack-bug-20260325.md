Those headers are not random. They look like your .S depth printer is doing something like:

emit('0' + depth)
So:

= = ASCII 61 = '0' + 13

a threaded interpreter continuation address

a CFA / dictionary pointer

a return address / saved IP

That makes this smell much more like an interpreter control-flow leak than arithmetic corruption.
